Private _clampLinesAccurate

  • _clampLinesAccurate(): void
    • Defined in packages/zent/src/clamp-lines/ClampLines.tsx:238

    This algorithm does not rely on the assumption that each line has same height. But it might be slow because its time complexity is proportional to text length and lines.

    It works by finding wrapping point(the postion where a line wrap happens) one by one.

    Note:

    Maually setting result string to DOM before return is required, because sometimes the setState just before return won't actually cause a re-render since there will cases when props.text changed but state.text remains the same as previous render.

    Returns void

Private _clampLinesFast

  • _clampLinesFast(): void
    • Defined in packages/zent/src/clamp-lines/ClampLines.tsx:173

    This algorithm relies on the assumption that each line has exactly the same height. The assumption does not always hold, especially when you mix different languages(e.g. CJK, emoji).

    It's fast because it uses binary search to find the overflow position.

    It should work well for small lines even if you have mixed languages in most cases.

    Note:

    Maually setting result string to DOM before return is required, because sometimes the setState just before return won't actually cause a re-render since there will cases when props.text changed but state.text remains the same as previous render.

    Returns void
